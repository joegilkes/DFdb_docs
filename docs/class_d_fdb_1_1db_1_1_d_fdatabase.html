Detailed Description

Wraps ASE database to add functionality.

Contains functions to ingest new structures and reaction paths, select rows in a database by the metadata tags applied to them, and re-export them to file.

Automatically attaches a new DFoptions instance using the default settings if an existing instance is not provided.

Parameters
db_filestr Path to the ASE database file.
modestr; Choices: (geometry, reaction) Mode to start the database in.
optionsDFoptions instance; Optional Pre-existing DFoptions instance, useful if transferring settings from one database to another without writing anything to file.

def DFdb.db.DFdatabase.add_batch (   self,
  traj_file,
  relaxed,
  eqm_struct = None 
)

Loads all structures in a single .traj file into a database. 

   Useful for adding all structures from a minima hopping run into
   the database, in which case "relaxed=False" should be interpreted
   as structures with their original inaccurate energies and forces,
   while "relaxed=True" means that the energies/forces have been updated
   to the level of accuracy needed from the respective method.

   In the case that the structures are from minima hopping, eqm_struct
   should be set to False so these structures are identifiable.

   The .traj file should start with "batch_", followed by the name of the
   structures contained within it.
Parameters
traj_filestr Path to ASE trajectory file containing relaxed structures. The filename should start with "batch_", followed by the name of the structures contained within it.
relaxedboolean Whether the structures are in their final state or require further calculation.
eqm_structboolean; Optional Whether this structure is a ground state structure, eg. from a geometry optimisation, or a non-equilibrium structure, eg. from minima hopping.

def DFdb.db.DFdatabase.add_reaction (   self,
  reac_file,
  nimages 
)

Load the end state of a NEB calculation into a database. 

   The final [nimages] of the provided trajectory will be loaded into
   the database and tagged under their reaction name, as well as a
   unique reaction id and their respective calculator.

   The transition state will be assumed to be the highest energy of
   the images, and will be marked as such with the `TS` tag.
Parameters
reac_filestr Path (not including path to reaction_folder in DFoptions) to NEB trajectory.
nimagesint Number of images in the band of the NEB calculation.
