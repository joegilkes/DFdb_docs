Detailed Description

Class which handles all database communication.

Contains functions to ingest new structures and reaction paths, identify those in need of optimisation, and to submit optimisations on these jobs.

Parameters

db_file Path to the ASE database file. options (optional) DFoptions instance.

Loads all structures in a single .traj file into the database. 

   Useful for adding all structures from a minima hopping run into
   the database, in which case "relaxed=False" should be interpreted
   as structures with their original inaccurate energies and forces,
   while "relaxed=True" means that the energies/forces have been updated
   to the level of accuracy needed from the respective method.

   In the case that the structures are from minima hopping, equ_struct
   should be set to False so these structures are identifiable.

   The .traj file should start with "batch_", followed by the name of the
   structures contained within it.
Parameters

traj_file Path to traj file containing relaxed structures. relaxed Boolean signifying whether the structures are in their final state or require further calculation. eqm_struct Boolean signifying if this structure is GS or non-equilibrium, eg. from minima hopping.

   The final [nimages] of the provided trajectory will be loaded into
   the database and tagged under their reaction name, as well as a

   The transition state will be assumed to be the highest energy of
   the images, and will be marked as such with the `TS` tag.
